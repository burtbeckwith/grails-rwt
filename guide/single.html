1 Introduction

This plugin integrates the RAP Widget Toolkit (RWT) - a web-based Ajax/JavaScript widget library similar to SWT - into your Grails application.

RWT is the UI part of the Rich Ajax Platform (RAP) - the web-ported Eclipse RCP application platform. RWT is provided in standalone mode by this plugin, without the full RAP workbench or OSGi. It also enables you to use the JFace API for building your user interfaces.

RWT enables you to create highly interactive, desktop-like Ajax interfaces by only writing Java/Groovy code. In contrast to the popular Google Web Toolkit (Grails plugin), RWT is not compiled to JavaScript, but provides its own set of JavaScript widget which communicate transparantly with your server-side code. Since basically everything happens on the server-side (besides the rendering in the browser), it is not necessary to use DTOs or other transfer objects to display the data on the client. The domain objects can be seamlessly used in the UI logic (e.g. via DataBinding).

For more information please refer to the documentation of the particular technology.

The following pages will provide you with the necessary information to get started with Grails and RWT. Have fun!

2 Getting Started

Getting started...

This plugin requires at least Grails version 2.0.

2.1 Installation

The plugin is installed using the default Grails plugin installation mechanism. Simply add the following lines to the plugin dependency section in the BuildConfig.groovy file:

grails.project.dependency.resolution = {
    …
    plugins {
        compile ':rwt:0.1'
    }
    …
}

2.2 Creating the Entry Point Bean

Create entry point ...

import org.eclipse.jface.viewers.ArrayContentProvider
import org.eclipse.jface.viewers.LabelProvider
import org.eclipse.jface.viewers.ListViewer
import org.eclipse.rap.rwt.lifecycle.IEntryPoint
import org.eclipse.swt.SWT
import org.eclipse.swt.events.SelectionAdapter
import org.eclipse.swt.events.SelectionEvent
import org.eclipse.swt.layout.GridData
import org.eclipse.swt.layout.GridLayout
import org.eclipse.swt.widgets.Button
import org.eclipse.swt.widgets.Display
import org.eclipse.swt.widgets.Label
import org.eclipse.swt.widgets.List
import org.eclipse.swt.widgets.Shell
class HelloEntryPointService implements IEntryPoint { // "The service needs to implement IEntryPoint"
    def fooService // "Other services will get injected by Grails"
    @Override
    int createUI() {
        Display display = new Display()
        Shell shell = new Shell(display)
        shell.setSize(500, 400)
        shell.layout = new GridLayout()
        Label label = new Label(shell, SWT.NONE)
        label.text = fooService.serviceMethod() // "Call methods of other services"
        // "JFace is available"
        ListViewer viewer = new ListViewer(shell, SWT.V_SCROLL | SWT.H_SCROLL)
        viewer.list.layoutData = new GridData(SWT.FILL, SWT.FILL, true, true)
        viewer.contentProvider = ArrayContentProvider.instance
        viewer.labelProvider = new LabelProvider()
        viewer.input = Foo.list() // "Use domain objects and GORM, no DTOs required!"
        Button button = new Button(shell, SWT.PUSH)
        button.layoutData = new GridData(SWT.BEGINNING, SWT.CENTER, false, false)
        button.text = "Hello!"
        button.addSelectionListener(new SelectionAdapter() {
            @Override
            public void widgetSelected(SelectionEvent e) {
                println "Hello RWT!"
            }
        })
        shell.open()
        return 0
    }
}

2.3 Configuring the Entry Point

Configure entry point ...

// The RWT configuration block
rwt {
    // Entry Point configuration block
    entrypoints {
        // Configuration of an entry point named 'hello'
        hello {
            bean = 'helloEntryPointService'
            pageTitle = 'Hello RWT!'
        }
    }
}

2.4 Running the Application

The application can be started in a local development setup using the default run-app Grails command from the command line in the project root folder:

grails run-app

After the application started you should be able to access your first Grails/RWT example under the following URL: http://localhost/foo/hello

2.5 Deploying the Application

You can now package your first Grails/RWT application into a WAR file and deploy to the servlet of your choice. The deployment procedure is the same as for any other Grails application, by simply issuing the following command in the command line:

grails war

The resulting WAR file is created in the target/ folder and can now be deployed to e.g. a Tomcat container. All necessary jar files for RWT/JFace will be contained and the web.xml is also properly set up.

3.3 Logging Configuration

The plugin logs some information in the INFO and DEBUG levels. To increase the verbosity of the log messages you can enable the debug level for some plugin artifacts. Add the following lines to the log4j section in your Config.groovy file:

log4j = {
    …
    debug 'grails.app.conf.RwtPluginBootStrap'
    debug 'grails.plugins.rwt'
    debug 'RwtGrailsPlugin'
    …
}

4 Advanced Topics

URL Mapping of Entry Points

By default RWT entry points are mapped to their name under the applications base URL. If the application name would be 'foo' and the entry point name 'hello', the URL would look like this: http://server/foo/hello

Sometimes it may be desired to reach the entry point under the base URL or a different URL. This can easily be achieved by using the Grails URL Mappings. To map the entry point to the base URL, the following needs to be configured in the UrlMappings.groovy file:

class UrlMappings {
    static mappings = {
        …
        "/"(uri: "/hello")
        …
    }
}

Now the RWT application is reachable via: http://server/foo
