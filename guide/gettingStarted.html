This plugin integrates RWT (standalone), the web-ported Standard Widget Toolkit (SWT), into your Grails application. It enables you to build your RWT and JFace powered user interface using Groovy and the complete Grails framework stack with all its features and syntactic DSL sugar.

2 Getting Started

Getting started…

2.1 Installation

The plugin is installed using the default Grails plugin installation mechanism. In the BuildConfig.groovy file simply add the following in the plugin dependency section:

grails.project.dependency.resolution = {
    …
    plugins {
        compile ':rwt:0.1'
    }
    …
}

2.2 Creating the Entry Point Bean

Create entry point ...

import org.eclipse.jface.viewers.ArrayContentProvider
import org.eclipse.jface.viewers.LabelProvider
import org.eclipse.jface.viewers.ListViewer
import org.eclipse.rap.rwt.lifecycle.IEntryPoint
import org.eclipse.swt.SWT
import org.eclipse.swt.events.SelectionAdapter
import org.eclipse.swt.events.SelectionEvent
import org.eclipse.swt.layout.GridData
import org.eclipse.swt.layout.GridLayout
import org.eclipse.swt.widgets.Button
import org.eclipse.swt.widgets.Display
import org.eclipse.swt.widgets.Label
import org.eclipse.swt.widgets.List
import org.eclipse.swt.widgets.Shell
class HelloEntryPointService implements IEntryPoint { // "The service needs to implement IEntryPoint"
    def fooService // "Other services will get injected by Grails"
    @Override
    int createUI() {
        Display display = new Display()
        Shell shell = new Shell(display)
        shell.setSize(500, 400)
        shell.layout = new GridLayout()
        Label label = new Label(shell, SWT.NONE)
        label.text = fooService.serviceMethod() // "Call methods of other services"
        // "JFace is available"
        ListViewer viewer = new ListViewer(shell, SWT.V_SCROLL | SWT.H_SCROLL)
        viewer.list.layoutData = new GridData(SWT.FILL, SWT.FILL, true, true)
        viewer.contentProvider = ArrayContentProvider.instance
        viewer.labelProvider = new LabelProvider()
        viewer.input = Foo.list() // "Use domain objects and GORM, no DTOs required!"
        Button button = new Button(shell, SWT.PUSH)
        button.layoutData = new GridData(SWT.BEGINNING, SWT.CENTER, false, false)
        button.text = "Hello!"
        button.addSelectionListener(new SelectionAdapter() {
            @Override
            public void widgetSelected(SelectionEvent e) {
                println "Hello RWT!"
            }
        })
        shell.open()
        return 0
    }
}

2.3 Configuring the Entry Point

Configure entry point ...

// The RWT configuration block
rwt {
    // Entry Point configuration block
    entrypoints {
        // Configuration of an entry point named 'hello'
        hello {
            bean = 'helloEntryPointService'
            pageTitle = 'Hello RWT!'
        }
    }
}

2.4 Running the Application

The application can be started in a local development setup using the default run-app Grails command from the command line in the project root folder:

grails run-app

After the application started you should be able to access your first RWT example under the following URL: http://localhost/foo/hello

2.5 Deploying the Application

You can now package your first Grails/RWT application into a WAR file and deploy to the servlet of your choice. The deployment procedure is the same as for any other Grails application, by simply issuing the following command in the command line:

grails war

The resulting WAR file is created in the target/ folder and can now be deployed to e.g. a Tomcat container. All necessary jar files for RWT/JFace will be contained and the web.xml is also properly set up.
